Propuesta: Proyecto KOALA

Evaluación Técnica y Funcional

  1. Servidor físico o virtualizado (a cargo del cliente)
    Ambas opciones son viables. Puede utilizarse un servidor físico o virtualizado, siempre que cuente con los recursos necesarios para alojar SQL Server Express y Power BI de forma conjunta.
  2. Uso de Microsoft SQL Server Express – fase inicial (a cargo del cliente)
    Es una alternativa válida para reducir los costos iniciales. Se debe considerar que tiene limitaciones de almacenamiento (10 GB por base), CPU y memoria. Esta elección es adecuada mientras se trabaje con datos de un único productor.
  3. SQL y Power BI en el mismo equipo – monopuesto (a cargo del cliente)
    Funciona correctamente en un entorno inicial. Se sugiere optimizar los recursos del equipo ya que se compartirán entre ambas aplicaciones. El acceso remoto deberá configurarse con seguridad adecuada.
  4. Fase 1 del Proyecto – Implementación ETL (a cargo de Fyrao)
    Se desarrollarán scripts en Python para la extracción, transformación y carga de datos del CRM Botmaker al nuevo diseño de tablas, según los distintos endpoints solicitados por el cliente. El proceso se automatizará cada 15 días. Esta etapa cubre a un único productor como muestra de validación.
  5. Visualización con Power BI (a cargo de Koala)
    Los datos migrados podrán ser visualizados con dashboards desarrollados por el propio equipo de Koala utilizando Power BI instalado en el servidor.
  6. Fase 2 propuesta – Reemplazo del CRM Botmaker (a cargo de Fyrao)
    Proponemos como Fase 2 un CRM propio, integrado a la base de datos ya migrada, lo que permitirá un mayor control de los datos, reportes más flexibles y continuidad histórica sin restricciones de Botmaker.
